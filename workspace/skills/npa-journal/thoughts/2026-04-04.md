## What I Got Wrong

### 1. JAM Rental-Only Misclassification
- **Mistake**: Treated JAM's 28,263 properties as mostly rentals
- **Reality**: 61% are FOR SALE — 11,350 units, 6,276 condos. JAM is the **2nd largest NPA source** after LED
- **Root cause**: Inherited assumption without verification. `type_sale_th` field clearly shows "ขาย" vs "เช่า"
- **What I should have done**: Query `SELECT type_sale_th, COUNT(*) FROM jam GROUP BY type_sale_th` on first encounter
- **Severity**: HIGH — we were ignoring the 2nd largest source of NPA deals
- **Status**: Corrected. JAM now fully integrated into cross-provider analysis.

### 2. Chewathai รามคำแหง Size Hallucination

## What I Got Right

### 1. Multi-Provider Comparison Framework
Building a side-by-side table of all 6 providers (count, BKK count, condos, strengths, best feature) gave immediate clarity on where to focus research time. This was the single most useful artifact of the session.

### 2. Hardcoded Haversine for Transit
When Overpass API hit 429 rate limits, falling back to hardcoded BTS/MRT station coordinates + haversine distance was instant and reliable. No API dependency, no rate limits. This should be the PRIMARY transit check method going forward.

### 3. Multi-Unit Building Discovery
Noticing that KTB had 4 units in the same building near MRT สุทธิสาร — and BAM had 2 units in Thru Thonglor — was valuable. When you find one good unit, checking for siblings in the same building reveals NPA concentrations.

### 4. Honest Discount Normalization
Recognizing that each provider defines "discount" differently (KTB = price/appraisal, KBANK = sell/promo, JAM = selling/discount, BAM = sell/shock) and calling this out explicitly prevents cross-provider comparison errors.

### 5. KB Ingestion Discipline
Every analysis block ended with KB ingestion with proper metadata (category, area, source). This builds a searchable knowledge base for future sessions.

### 6. Yesterday's Lesson Applied
Applied yesterday's journal corrections:
- ✅ Verified data units (KTB `area` is text "XX ตร.ม." — caught before calculation)
- ✅ Showed LOW/MID/HIGH rent scenarios implicitly via market comparables
- ✅ Auto-skepticism for >50% discounts (SAM 15 flagged, KTB 62% flagged with "WHY?" note)
- ✅ Identified building/project before analysis (JAM 315 = ไลฟ์ รัชดาภิเษก, etc.)
